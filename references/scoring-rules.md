# GEO诊断报告 - 六维评分规则

## 概述

GEO（Generative Engine Optimization）诊断采用六维评分模型，每个维度独立评分后按权重加权汇总，得出综合得分。每个维度满分100分，综合满分100分。

---

## 维度一：品牌提及率（Brand Mention）

### 计算公式

```
品牌提及率 = (品牌被正确提及的问题数 / 总测试问题数) x 100
```

### 评分细则

| 提及情况 | 得分 | 说明 |
|---------|------|------|
| 品牌被正确提及且为主要推荐 | 90-100 | AI在回答中主动提及品牌并作为核心推荐 |
| 品牌被提及但非主要推荐 | 60-89 | 品牌出现在回答中，但非首选推荐 |
| 品牌未被提及但竞品也未提及 | 30-59 | 品牌未被提及，但该问题领域无明确竞品 |
| 品牌未被提及且竞品被提及 | 0-29 | 竞品被推荐，品牌完全缺失 |

### 权重

**0.20**（综合得分占比20%）

---

## 维度二：回答准确度（Answer Accuracy）

### 计算公式

```
准确度 = (信息准确的问题数 / 总测试问题数) x 100
```

### 评分细则

| 准确程度 | 得分 | 说明 |
|---------|------|------|
| 完全准确 | 90-100 | AI回答中的品牌信息与官方一致，无事实错误 |
| 基本准确 | 70-89 | 核心信息正确，细节存在轻微偏差 |
| 部分准确 | 40-69 | 关键信息有误或过时，但不影响整体判断 |
| 严重失实 | 0-39 | 包含重大事实错误或误导性信息 |

### 权重

**0.20**（综合得分占比20%）

---

## 维度三：来源引用质量（Source Citation）

### 计算公式

```
引用质量 = (官方来源引用次数 / 总引用次数) x 100
```

### 评分细则

| 引用质量 | 得分 | 说明 |
|---------|------|------|
| 优质引用 | 90-100 | AI引用来源以官网、权威媒体、官方文档为主 |
| 良好引用 | 70-89 | 官方来源占多数，少量第三方来源 |
| 一般引用 | 40-69 | 第三方来源为主，官方来源较少 |
| 劣质引用 | 0-39 | 引用来源不可靠或无引用 |

### 权重

**0.15**（综合得分占比15%）

---

## 维度四：内容结构化程度（Content Structure）

### 计算公式

```
结构化程度 = (达标结构项数 / 总检查项数) x 100
```

### 检查项清单

| 检查项 | 达标标准 | 权重 |
|--------|---------|------|
| 标题层级 | 使用H1-H6语义化标签，层级清晰 | 15% |
| 结构化数据 | 包含Schema.org标记（Organization、Product等） | 20% |
| FAQ格式 | 使用FAQPage结构化数据 | 15% |
| 内容分块 | 内容按主题分块，段落长度适中（<300字） | 15% |
| 列表使用 | 关键信息使用有序/无序列表呈现 | 10% |
| 内链结构 | 核心页面间有合理的内链关系 | 10% |
| 元数据完整 | Title、Description、OG标签完整准确 | 15% |

### 评分细则

| 结构化程度 | 得分 | 说明 |
|-----------|------|------|
| 优秀 | 90-100 | 7项全部达标，且结构化数据丰富 |
| 良好 | 70-89 | 5-6项达标，核心结构化数据完备 |
| 一般 | 40-69 | 3-4项达标，存在明显结构缺陷 |
| 较差 | 0-39 | 不足3项达标，内容缺乏结构化 |

### 权重

**0.15**（综合得分占比15%）

---

## 维度五：知识库完善度（Knowledge Base）

### 计算公式

```
知识库完善度 = (已覆盖内容类型数 / 应覆盖内容类型总数) x 100
```

### 内容类型覆盖检查

根据行业配置中的 `content_types` 列表，检查每类内容的覆盖情况：

| 覆盖状态 | 单项得分 | 说明 |
|---------|---------|------|
| 完整覆盖 | 100 | 内容完整、准确、近期更新 |
| 部分覆盖 | 60 | 内容存在但不完整或需要更新 |
| 未覆盖 | 0 | 完全缺失该类型内容 |

### 评分细则

| 完善度 | 得分 | 说明 |
|--------|------|------|
| 优秀 | 90-100 | P0内容100%覆盖，P1内容>80%覆盖 |
| 良好 | 70-89 | P0内容100%覆盖，P1内容>60%覆盖 |
| 一般 | 40-69 | P0内容>80%覆盖，P1内容<60%覆盖 |
| 较差 | 0-39 | P0内容<80%覆盖 |

### 权重

**0.15**（综合得分占比15%）

---

## 维度六：渠道覆盖度（Channel Coverage）

### 计算公式

```
渠道覆盖度 = (有效渠道数 / 目标渠道总数) x 100
```

### 渠道分层评估

| 层级 | 渠道类型 | 评估标准 |
|------|---------|---------|
| L1 核心层 | 官网、官方公众号、官方视频号 | 必须全部覆盖且活跃 |
| L2 扩展层 | 知乎、百家号、CSDN、小红书 | 至少覆盖3个且内容更新 |
| L3 影响层 | 行业媒体、评测网站、论坛 | 至少覆盖2个 |
| L4 长尾层 | 其他UGC平台、问答平台 | 尽可能覆盖 |

### 评分细则

| 覆盖程度 | 得分 | 说明 |
|---------|------|------|
| 优秀 | 90-100 | L1全覆盖，L2覆盖>=4，L3>=3，L4>=2 |
| 良好 | 70-89 | L1全覆盖，L2覆盖>=3，L3>=2 |
| 一般 | 40-69 | L1全覆盖，L2覆盖>=2，L3>=1 |
| 较差 | 0-39 | L1未全覆盖或L2覆盖<2 |

### 权重

**0.15**（综合得分占比15%）

---

## 综合得分计算

### 公式

```
综合得分 = 品牌提及率 x 0.20
         + 回答准确度 x 0.20
         + 来源引用质量 x 0.15
         + 内容结构化程度 x 0.15
         + 知识库完善度 x 0.15
         + 渠道覆盖度 x 0.15
```

### 等级映射

| 综合得分 | 等级 | 标签颜色 | 说明 |
|---------|------|---------|------|
| 90-100 | S | green | GEO表现优秀，品牌在AI搜索中占据主导 |
| 80-89 | A | green | GEO表现良好，仅需微调优化 |
| 70-79 | B | blue | GEO表现中等，存在明显提升空间 |
| 60-69 | C | yellow | GEO表现较差，需要系统性优化 |
| 0-59 | D | red | GEO表现极差，需要从基础开始建设 |

### 各维度等级映射

| 得分区间 | 等级 | CSS Class |
|---------|------|-----------|
| 90-100 | 优秀 | excellent |
| 75-89 | 良好 | good |
| 60-74 | 一般 | fair |
| 0-59 | 较差 | poor |

---

## 评分使用说明

1. 各维度评分基于实际测试数据，非主观判断
2. 行业配置中的 `scoring_weights` 可调整各维度权重
3. 综合得分保留一位小数
4. 等级标签对应HTML模板中的 `.tag` 颜色类名
